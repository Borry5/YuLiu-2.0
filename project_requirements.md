# 语流 (YuLiu) - Flowchart-Driven AI Dialogue System
## 项目需求说明书 (Project Requirements Document)

### 1. 项目概述
**项目名称**: 语流 (YuLiu)
**核心理念**: 解决传统线性对话模式在处理复杂任务时的上下文丢失和迷失感问题。通过可视化的流程图与对话相结合，让用户清晰感知当前任务进度、分支选择和待解决问题。
**目标用户**: 需要利用 AI 辅助完成复杂任务（如编程、写作、学习新知识）的用户。
**部署方式**: 本地运行 Web 应用 (Localhost)，保留未来在线部属的可能性。

### 2. 核心功能及需求

#### 2.1 双视图交互界面
- **界面布局**: 左侧/右侧分栏结构。
    - **对话区**: 传统的 Chat Interface，用于输入指令和接收文本回复。
    - **流程图区**: 动态生成的思维导图/流程图，实时反映对话逻辑。
- **联动机制**:
    - 点击流程图节点，对话区自动跳转到相关上下文历史。
    - 对话中提到的关键步骤，自动在流程图中生成新节点。

#### 2.2 智能分支管理 (Smart Branching)
- **主干流程**: 代表任务的主要推进路径（例如：问题分析 -> 方案提出 -> 执行步骤 1 -> 执行步骤 2）。
- **主要分支 (Major Branch)**: 当用户需要尝试不同方案或回溯重做时（例如：“方案 A 失败，尝试方案 B”），生成平行分支。支持用户在不同分支间切换，上下文随之切换。
- **次要分支/详情 (Minor Branch/Detail)**: 针对主干内容的补充提问（例如：“什么是中间代码？”）。
    - **UI 表现**: 不应作为平行庞大分支干扰主线，而是作为主干节点的“侧边注脚”或“展开详情”。
    - **交互**: 类似 Notion 的 Toggle 或 Side-peek，阅读完后可收起，继续主线任务。

#### 2.3 上下文感知与记忆 (Context Awareness)
- **实时状态显示**: 流程图需高亮显示“当前执行阶段”。
- **自动摘要**:
    - 当用户切换话题或长时间未操作后，AI 自动生成“目前进度摘要”。
    - 切换分支时，AI 能够快速读取该分支的历史并生成简报。
- **新话题检测**: 如果用户输入与当前流程无关的内容，系统提示是否新建对话列表或将其作为新分支处理。

#### 2.4 知识库与文档支持
- **文件上传**: 支持 PDF、PPT 等文档拖拽上传。
- **智能引用**: AI 回答时需标注来源（例如：“根据第三章第四节...”），并支持点击引用跳转到文档对应位置（或在对话框旁展示预览）。
- **无需频繁滚动**: 提问文档细节时，通过“次要分支”或“悬浮窗”展示，避免在长对话中反复上下翻阅。

#### 2.5 系统配置与安全性
- **API Key 管理**: 用户自行输入 API Key（首发支持 DeepSeek、SiliconFlow）。
- **本地存储**: 对话记录、流程图结构、API Key 均存储在本地浏览器 (LocalStorage/IndexedDB)，保障隐私。

### 3. 技术栈选型 (Technology Stack)
- **前端框架**: React (成熟、生态丰富)
- **构建工具**: Vite (快速、轻量)
- **UI 组件库**: TailwindCSS + Shadcn/UI (现代化、美观)
- **流程图引擎**: React Flow (专为 React 设计的节点流程库，功能强大)
- **状态管理**: Zustand (轻量级，适合复杂状态同步)
- **图标库**: Lucide React
- **Markdown 渲染**: React-Markdown

### 4. 界面设计风格 (UI/UX)
- **风格**: 极简、现代、科技感。
- **主题**: 支持深色/浅色模式切换。
- **配色**: 使用柔和的渐变色区分不同类型的节点（已完成、进行中、待解决、分支）。
- **动效**: 节点生成、连线过程需有平滑的过渡动画，减少突兀感。
